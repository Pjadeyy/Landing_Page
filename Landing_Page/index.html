Project Overview

A Final Project Designed to Make
Wedding Planning Easier

HJ Wedding Planner is a mobile-focused wedding planning system that helps couples compare packages, organize tasks, communicate with coordinators, and monitor booking progress in one guided workflow.

Purpose & Value

One organized platform instead of scattered chats, notes, and booking updates.

Our project solves a common problem for couples and coordinators: planning information is often spread across multiple apps and conversations. HJ Wedding Planner centralizes package browsing, task management, status tracking, and direct communication in a single mobile experience.

Target users: Couples, coordinators, and event teams
Main goal: Reduce planning stress through guided organization
Outcome: Clearer decisions, faster coordination, better visibility

Technical Details

Built with a Clear Stack,
Architecture, and Process

This section summarizes the technologies used in the landing page and mobile application, the high-level architecture behind the system, and the development workflow followed by the team.

Tech Stack

  • Landing page built with HTML5, CSS3, and vanilla JavaScript
  • Mobile application designed for Android implementation
  • Java development workflow supported in Android Studio
  • Firebase-ready backend support for data, sync, and messaging

Architecture

  • Presentation layer for onboarding, package browsing, and dashboard views
  • Application logic for checklist handling, booking flow, and notifications
  • Cloud-connected data layer for real-time chat, user records, and status updates
  • Modular structure that separates interface, business logic, and project resources

Development Process

  • Requirement gathering based on wedding planning pain points and user needs
  • UI planning through wireframes, screen flow, and content mapping
  • Feature implementation for packages, messaging, checklist, and progress tracking
  • Iterative refinement through testing, feedback, and documentation updates

Testing & Quality

  • QA review for usability, clarity, and interface consistency
  • Responsive checks across desktop and mobile screen widths
  • Content validation for links, sections, and documentation access
  • Final polishing for presentation, readability, and smooth interaction
