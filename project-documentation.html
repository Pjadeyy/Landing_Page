Project Overview

Project Overview

HJ Wedding Planner Mobile Application is a wedding planning platform designed to make coordination faster, more organized, and easier for both clients and coordinators. It combines package browsing, gown selection, booking, appointment tracking, and direct communication in one mobile-focused system.

This project was prepared as part of the final term major activity for IT-1310 Application Development and Emerging Technologies.

Purpose and Audience

Why the Project Matters

  • It solves the problem of scattered wedding planning information across chats, social media, and manual records.
  • It supports clients who need clearer booking, scheduling, and package information.
  • It supports coordinators who need better organization, communication, and appointment handling.
  • It is intended for developers, designers, testers, project stakeholders, and the proposed client.
Problem Statement

Current Challenge

Wedding planning often depends on manual scheduling and social media messaging. That setup can create delayed responses, disorganized records, overlapping schedules, and miscommunication between clients and coordinators.

The project addresses this with a centralized application for bookings, appointments, records, and messaging.

Objectives

Project Goals

  • Gather functional requirements from the client based on real-world needs.
  • Design the system architecture, interface, and workflow structure.
  • Develop the application through Android Studio using proper programming practices.
  • Test and improve reliability, usability, and overall system quality.
User Features

Core Client Functions

  • User registration and authentication with email/password and Google Sign-In.
  • Viewing of wedding packages, pricing, inclusions, and service options.
  • Browsing of gown collections with design and style details.
  • Booking and appointment scheduling for chosen services and dates.
  • Real-time or near real-time communication with coordinators.
  • Calendar integration for tracking appointments and wedding deadlines.
  • Google Maps support for location assistance.
Admin Features

Coordinator and Management Functions

  • Manage wedding packages, pricing, inclusions, and availability.
  • Manage gown collections by adding, updating, and removing entries.
  • Monitor bookings and appointments and resolve schedule conflicts.
  • Handle client concerns and respond through the communication system.
  • Organize schedules for appointments, events, and coordination tasks.
  • Manage user accounts and system-side concerns securely.
Technical Details

Technical Details

  • Platform: Android mobile application.
  • Development Tool: Android Studio.
  • Backend Direction: Firebase services for authentication, synchronization, and messaging.
  • Architecture: MVVM-based separation of user interface, logic, and data handling.
  • Integrations: Google Sign-In, calendar tracking, and Google Maps support.
Development Process

How the Project Was Built

  • Requirements gathering from the client and project stakeholders.
  • UI/UX planning through wireframes, mockups, and workflow design.
  • Feature implementation for authentication, booking, messaging, and browsing.
  • Testing for usability, responsiveness, reliability, and error handling.
Architecture and Data

System Structure

  • The project includes admin and user data flow diagrams in the source documentation.
  • The database uses a unified users table with role-based access instead of separate user tables.
  • Modules are linked using fields such as created_by, approved_by, responded_by, and managed_by.
  • This structure supports scalability, maintainability, and cleaner user management.
Roles and Devices

Implementation Notes

  • User role supports package browsing, gown viewing, booking, and inquiries.
  • Administrator role supports package, booking, schedule, and account management.
  • The system targets Android devices and adapts to different screen sizes and resolutions.
  • A stable internet connection is needed for Firebase-backed real-time features.
Security and Privacy

Protection of User Data

  • Authentication supports email/password and Google Sign-In.
  • Authorization is role-based and enforced for protected features.
  • HTTPS is used for secure communication.
  • Stored credentials are protected through hashing or secure backend-managed storage.
  • The documentation aligns the project with the Data Privacy Act of 2012 in the Philippines.
Accessibility and Usability

Design Quality Notes

  • Readable fonts and consistent spacing improve clarity.
  • Touch-friendly controls reduce mobile input errors.
  • Clear navigation supports browsing, booking, chat, and schedule tracking.
  • Error prompts and user feedback help guide task completion.

Glossary Highlights

Key Technical Terms

  • API: rules that allow software systems to communicate.
  • Authentication: verifying the identity of a user.
  • Authorization: controlling what a verified user can do.
  • Firebase: cloud platform for authentication, database, and storage services.
  • MVVM: software architecture that separates UI, logic, and data handling.
  • RBAC: role-based access control for system permissions.
